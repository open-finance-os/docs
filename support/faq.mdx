# Frequently Asked Questions

Quick answers to common questions about Open Finance development and the hackathon.

## General Questions

<AccordionGroup>
  <Accordion title="What is Open Finance?">
    Open Finance is a framework that enables secure, standardized access to financial data and services through APIs. It allows third-party developers to build innovative financial applications with user consent.

    Key features:
    - Standardized APIs for banking, insurance, and payments
    - Secure authentication and authorization
    - User consent management
    - Real-time data access
  </Accordion>

  <Accordion title="How do I get started with the Open Finance sandbox?">
    Follow these steps:

    1. Register at the sandbox portal
    2. Obtain your API credentials
    3. Review the [authentication guide](/api-reference/introduction)
    4. Test with sample API calls
    5. Build your application

    See our [quickstart guide](/quickstart) for detailed instructions.
  </Accordion>

  <Accordion title="What programming languages can I use?">
    You can use any programming language that supports HTTP requests. Popular choices include:

    - **JavaScript/TypeScript** (Node.js, React)
    - **Python** (FastAPI, Django)
    - **Java** (Spring Boot)
    - **Go**
    - **C#** (.NET)

    We provide examples in JavaScript and Python, but the APIs are language-agnostic.
  </Accordion>
</AccordionGroup>

## API & Technical Questions

<AccordionGroup>
  <Accordion title="How do I authenticate with the Open Finance APIs?">
    The APIs use OAuth 2.0 for authentication:

    ```javascript
    const getToken = async () => {
      const response = await fetch('/oauth/token', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          client_id: 'YOUR_CLIENT_ID',
          client_secret: 'YOUR_CLIENT_SECRET',
          grant_type: 'client_credentials'
        })
      });

      return response.json();
    };
    ```

    Token must be included in all API requests:
    ```javascript
    headers: {
      'Authorization': `Bearer ${token}`
    }
    ```
  </Accordion>

  <Accordion title="What are the API rate limits?">
    Sandbox environment limits:
    - **Standard endpoints**: 100 requests per minute
    - **Data-intensive endpoints**: 20 requests per minute
    - **Bulk operations**: 5 requests per minute

    Implement exponential backoff for rate limit errors (429 status code).
  </Accordion>

  <Accordion title="How do I handle API errors?">
    Common error codes and handling:

    | Code | Meaning | Action |
    |------|---------|--------|
    | 400 | Bad Request | Check request format |
    | 401 | Unauthorized | Refresh authentication token |
    | 403 | Forbidden | Check permissions/scope |
    | 404 | Not Found | Verify endpoint/resource |
    | 429 | Too Many Requests | Implement backoff |
    | 500 | Server Error | Retry with exponential backoff |

    Example error handling:
    ```javascript
    try {
      const response = await fetch(url);
      if (!response.ok) {
        throw new Error(`API Error: ${response.status}`);
      }
      return response.json();
    } catch (error) {
      console.error('Request failed:', error);
      // Implement retry logic here
    }
    ```
  </Accordion>

  <Accordion title="Can I test with mock data?">
    Yes! The sandbox provides mock data for all endpoints. Enable mock mode:

    ```javascript
    const config = {
      baseUrl: 'https://sandbox.openfinance.ae/api/v1',
      useMockData: true,
      mockDelay: 500 // Simulate network delay
    };
    ```

    Mock data includes realistic:
    - Account information
    - Transaction histories
    - Insurance policies
    - Payment scenarios
  </Accordion>
</AccordionGroup>

## Claude Code Questions

<AccordionGroup>
  <Accordion title="What is Claude Code and how can it help?">
    Claude Code is an AI-powered development assistant that can:

    - Generate API integration code
    - Debug authentication issues
    - Create data models from API responses
    - Write test cases
    - Explain complex financial concepts
    - Optimize your code

    See our [development guide](/development) for AI-enhanced development setup.
  </Accordion>

  <Accordion title="How do I configure Claude Code for Open Finance development?">
    1. Install Claude Code following [the development guide](/development)
    2. Configure with Open Finance context:

    ```bash
    # Create a context file
    echo "I'm building with Open Finance APIs" > .claude-context

    # Add API documentation reference
    claude-code config --docs "https://docs.openfinance.ae"
    ```

    3. Use Claude Code commands:
    - Generate API client: `@claude create API client for banking endpoints`
    - Debug issues: `@claude why is my authentication failing?`
    - Optimize code: `@claude optimize this API integration`
  </Accordion>

  <Accordion title="Can Claude Code help with hackathon submissions?">
    Yes! Claude Code can assist with:

    - **Documentation**: Generate comprehensive README files
    - **Code Quality**: Refactor and optimize your code
    - **Testing**: Create unit and integration tests
    - **Architecture**: Suggest design patterns and structure
    - **Demo Scripts**: Help prepare presentation materials

    Example prompt:
    ```
    @claude help me create a README for my Open Finance hackathon project that uses banking APIs to provide financial insights
    ```
  </Accordion>
</AccordionGroup>

## Hackathon Specific Questions

<AccordionGroup>
  <Accordion title="What are the judging criteria?">
    Projects are evaluated on:

    1. **Innovation (30%)**: Creativity and uniqueness of solution
    2. **Technical Implementation (25%)**: Code quality, architecture, API usage
    3. **User Experience (20%)**: Interface design, usability
    4. **Business Potential (15%)**: Market viability, scalability
    5. **Presentation (10%)**: Demo quality, documentation

    Focus on building a working prototype that demonstrates your idea clearly.
  </Accordion>

  <Accordion title="Can I use external libraries and services?">
    Yes, you can use:
    - ✅ Open source libraries and frameworks
    - ✅ UI component libraries
    - ✅ Database services
    - ✅ Cloud hosting platforms
    - ✅ Analytics tools

    Must disclose:
    - All third-party services used
    - Any pre-existing code
    - External APIs beyond Open Finance

    Cannot use:
    - ❌ Proprietary/paid APIs without disclosure
    - ❌ Code written before hackathon (except boilerplate)
    - ❌ Others' work without attribution
  </Accordion>

  <Accordion title="What should my submission include?">
    Required components:

    1. **Source Code**
       - GitHub repository (public or private with access)
       - Clear folder structure
       - `.env.example` file

    2. **Documentation**
       - README with setup instructions
       - Architecture diagram
       - API endpoints used

    3. **Demo**
       - Live demo URL (if web-based)
       - Video walkthrough (2-3 minutes)
       - Screenshots

    4. **Presentation**
       - Pitch deck (5-10 slides)
       - Use case explanation
       - Future roadmap
  </Accordion>

  <Accordion title="How do I handle sensitive data?">
    Security best practices:

    ```javascript
    // DON'T commit sensitive data
    const apiKey = "sk_live_abc123"; // ❌ Never hardcode

    // DO use environment variables
    const apiKey = process.env.API_KEY; // ✅ Secure

    // DO validate and sanitize inputs
    const sanitizedInput = validator.escape(userInput);

    // DO use HTTPS for all API calls
    const response = await fetch('https://...'); // ✅

    // DON'T log sensitive information
    console.log(`User password: ${password}`); // ❌
    console.log('Authentication successful'); // ✅
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="My API calls are returning CORS errors">
    CORS errors in browser-based applications can be resolved by:

    1. **Development**: Use a proxy
    ```javascript
    // In package.json (React)
    "proxy": "https://sandbox.openfinance.ae"
    ```

    2. **Production**: Configure server-side proxy
    ```javascript
    // Express.js example
    app.use('/api', createProxyMiddleware({
      target: 'https://sandbox.openfinance.ae',
      changeOrigin: true
    }));
    ```

    3. **Alternative**: Make API calls from your backend
  </Accordion>

  <Accordion title="I'm getting 'Invalid Token' errors">
    Common causes and fixes:

    1. **Token expired**: Implement refresh logic
    ```javascript
    if (error.status === 401) {
      const newToken = await refreshToken();
      // Retry request with new token
    }
    ```

    2. **Wrong environment**: Verify sandbox vs production
    3. **Incorrect format**: Ensure "Bearer " prefix
    4. **Scope issues**: Check required permissions
  </Accordion>

  <Accordion title="The sandbox is slow or unresponsive">
    Try these solutions:

    1. **Check status page** for maintenance windows
    2. **Implement caching** to reduce API calls
    3. **Use mock data** during development
    4. **Batch requests** where possible
    5. **Add retry logic** with exponential backoff

    ```javascript
    const fetchWithRetry = async (url, retries = 3) => {
      for (let i = 0; i < retries; i++) {
        try {
          return await fetch(url);
        } catch (error) {
          if (i === retries - 1) throw error;
          await new Promise(r => setTimeout(r, 2 ** i * 1000));
        }
      }
    };
    ```
  </Accordion>
</AccordionGroup>
