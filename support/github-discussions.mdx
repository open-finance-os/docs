# Using GitHub Discussions for Support

GitHub Discussions is our primary support channel for the Open Finance community and hackathon participants. It's a collaborative space where you can ask questions, share ideas, and help others.


## Creating a New Discussion

### Before Posting

<Steps>
  <Step title="Search Existing Discussions">
    Use the search bar to check if your question has already been answered
  </Step>

  <Step title="Choose the Right Category">
    Select the most appropriate category for your discussion
  </Step>

  <Step title="Prepare Your Question">
    Include relevant details:
    - What you're trying to achieve
    - Steps you've already taken
    - Error messages (if any)
    - Code snippets (use markdown formatting)
  </Step>
</Steps>

### Writing a Good Discussion Post

<Tabs>
  <Tab title="Title">
    Make it descriptive and specific:
    - ‚úÖ Good: "Error 401 when calling Banking API /accounts endpoint"
    - ‚ùå Bad: "API not working"
  </Tab>

  <Tab title="Description">
    Include essential information:
    ```markdown
    ## Problem Description
    Brief explanation of the issue

    ## Steps to Reproduce
    1. First step
    2. Second step
    3. Error occurs

    ## Expected Behavior
    What should happen

    ## Actual Behavior
    What actually happens

    ## Environment
    - OS: [e.g., macOS, Windows, Linux]
    - Claude Code version: [if applicable]
    - API version: [if known]
    ```
  </Tab>

  <Tab title="Code Examples">
    Share relevant code with proper formatting:
    ````markdown
    ```javascript
    // Your code here
    const response = await fetch('/api/accounts');
    ```
    ````
  </Tab>
</Tabs>

## Getting Faster Responses

<Info>
  **Pro Tips for Quick Help:**
  1. Use descriptive titles
  2. Tag relevant team members or mentors
  3. Include minimal reproducible examples
  4. Respond to clarification requests promptly
  5. Check discussions during active hours
</Info>

## Contributing to Discussions

Help build our community by:
- Answering questions you know the solution to
- Sharing helpful resources and documentation
- Upvoting helpful discussions and comments
- Providing constructive feedback on ideas

## Moderation Guidelines

Our discussions follow these principles:
- **Be Respectful**: Treat everyone with courtesy
- **Stay On Topic**: Keep discussions relevant to Open Finance
- **No Spam**: Avoid promotional or off-topic content
- **Protect Privacy**: Never share personal or sensitive data

<Warning>
  Violations of community guidelines may result in post removal or account restrictions.
</Warning>
