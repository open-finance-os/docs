## Essential Development Setup

<Steps>
  <Step title="Install Core Tools">
    ### Required Tools

    ```bash
    # Check Node.js version
    node --version  # Should be 19+

    # Install Git (if not already installed)
    # Visit: https://git-scm.com/downloads

    # Install VS Code (recommended)
    # Visit: https://code.visualstudio.com/
    ```

    ### Recommended VS Code Extensions

    ```bash
    # Install useful extensions
    code --install-extension ms-vscode.vscode-typescript-next
    code --install-extension bradlc.vscode-tailwindcss
    code --install-extension ms-vscode.vscode-json
    code --install-extension redhat.vscode-yaml
    ```
  </Step>

  <Step title="Set Up Open Finance Environment">
    ### Create Project Structure

    ```bash
    # Create your hackathon project
    mkdir uae-hackathon-project
    cd uae-hackathon-project

    # Initialize Git repository
    git init
    git branch -M main

    # Create basic structure
    mkdir src api docs
    touch .env .gitignore README.md
    ```

    ### Configure Environment Variables

    Add to `.env`:
    ```bash
    # Open Finance API Configuration
    OPENFINANCE_CLIENT_ID=your_client_id
    OPENFINANCE_CLIENT_SECRET=your_client_secret
    OPENFINANCE_API_KEY=your_api_key
    OPENFINANCE_BASE_URL=https://sandbox.openfinance.ae/api/v1

    # Development Settings
    NODE_ENV=development
    PORT=3000
    ```

    Add to `.gitignore`:
    ```bash
    # Environment variables
    .env
    .env.local

    # Dependencies
    node_modules/
    npm-debug.log*

    # Build outputs
    dist/
    build/

    # IDE files
    .vscode/
    .idea/

    # OS files
    .DS_Store
    Thumbs.db
    ```
  </Step>

  <Step title="Install AI Development Tools (Optional)">

    </Tip>
  </Step>
</Steps>

## Framework-Specific Setup

Choose your development stack:

<Tabs>
  <Tab title="React/Next.js">
    ```bash
    # Create Next.js app
    npx create-next-app@latest my-openfinance-app
    cd my-openfinance-app

    # Install Open Finance dependencies
    npm install axios @types/axios
    npm install -D @types/node

    # Start development server
    npm run dev
    ```

    ### Sample API Integration
    ```javascript
    // lib/openfinance.js
    import axios from 'axios';

    const api = axios.create({
      baseURL: process.env.OPENFINANCE_BASE_URL,
      headers: {
        'Content-Type': 'application/json',
      },
    });

    export default api;
    ```
  </Tab>

  <Tab title="Node.js/Express">
    ```bash
    # Initialize Node.js project
    npm init -y

    # Install backend dependencies
    npm install express cors dotenv axios
    npm install -D nodemon

    # Update package.json scripts
    # "dev": "nodemon server.js"
    # "start": "node server.js"
    ```

    ### Sample Server Setup
    ```javascript
    // server.js
    const express = require('express');
    const cors = require('cors');
    require('dotenv').config();

    const app = express();
    const PORT = process.env.PORT || 3000;

    app.use(cors());
    app.use(express.json());

    // Health check
    app.get('/health', (req, res) => {
      res.json({ status: 'OK', timestamp: new Date().toISOString() });
    });

    app.listen(PORT, () => {
      console.log(`Server running on port ${PORT}`);
    });
    ```
  </Tab>

  <Tab title="Python/FastAPI">
    ```bash
    # Create virtual environment
    python -m venv hackathon-env
    source hackathon-env/bin/activate  # On Windows: hackathon-env\Scripts\activate

    # Install dependencies
    pip install fastapi uvicorn requests python-dotenv

    # Create requirements.txt
    pip freeze > requirements.txt
    ```

    ### Sample FastAPI Setup
    ```python
    # main.py
    from fastapi import FastAPI
    from fastapi.middleware.cors import CORSMiddleware
    import os
    from dotenv import load_dotenv

    load_dotenv()

    app = FastAPI(title="Open Finance Hackathon API")

    app.add_middleware(
        CORSMiddleware,
        allow_origins=["*"],
        allow_methods=["*"],
        allow_headers=["*"],
    )

    @app.get("/health")
    def health_check():
        return {"status": "OK"}

    # Run with: uvicorn main:app --reload
    ```
  </Tab>
</Tabs>

## Testing Your Setup

### Verify API Connection

```javascript
// test-connection.js
require('dotenv').config();
const axios = require('axios');

async function testConnection() {
  try {
    const response = await axios.post(
      `${process.env.OPENFINANCE_BASE_URL}/oauth/token`,
      {
        client_id: process.env.OPENFINANCE_CLIENT_ID,
        client_secret: process.env.OPENFINANCE_CLIENT_SECRET,
        grant_type: 'client_credentials'
      }
    );

    console.log('✅ Connection successful!');
    console.log('Token received:', response.data.access_token.substring(0, 20) + '...');
  } catch (error) {
    console.log('❌ Connection failed:', error.message);
  }
}

testConnection();
```

Run the test:
```bash
node test-connection.js
```

## Development Best Practices

### Code Organization

```
your-project/
├── src/
│   ├── components/     # UI components
│   ├── services/       # API services
│   ├── utils/          # Helper functions
│   └── types/          # Type definitions
├── api/
│   ├── auth.js         # Authentication logic
│   ├── accounts.js     # Account APIs
│   └── payments.js     # Payment APIs
├── docs/
│   └── api.md          # API documentation
├── .env                # Environment variables
├── .gitignore          # Git ignore rules
└── README.md           # Project documentation
```

### Security Checklist

<AccordionGroup>
  <Accordion title="✅ Environment Variables" icon="shield-check">
    - Never commit `.env` files
    - Use different credentials for dev/prod
    - Validate all environment variables on startup
  </Accordion>

  <Accordion title="✅ API Security" icon="lock">
    - Always use HTTPS in production
    - Implement proper error handling
    - Don't expose sensitive data in logs
    - Validate and sanitize all inputs
  </Accordion>

  <Accordion title="✅ Code Quality" icon="code">
    - Use TypeScript for better type safety
    - Implement proper error boundaries
    - Add comprehensive logging
    - Write tests for critical functions
  </Accordion>
</AccordionGroup>
