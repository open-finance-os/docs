A powerful AI-powered coding assistant that runs locally through Amazon Bedrock, providing intelligent code generation, analysis, and debugging capabilities for your hackathon projects.

Claude Code is an interactive CLI tool that helps developers with software engineering tasks. For this hackathon, access to Claude via Amazon Bedrock is pre-configured and provided through a bearer token‚Äîno AWS account setup or CLI installation required. Simply install Claude Code, configure your token, and start building!

With Claude Code, you can generate code, debug issues, refactor existing code, explain complex logic, and automate repetitive tasks‚Äîall from your terminal.

## Prerequisites

Before starting, ensure you have:

- **Node.js** (v18 or later) and npm installed
- **AWS Bearer Token** provided by ADCB
- A code editor and terminal

## Setting up Claude Code via Bedrock

### Step 1: Verify Node.js Installation

Check that Node.js is installed:

```bash
node --version
npm --version
```

If not installed, download from [nodejs.org](https://nodejs.org/) (LTS version recommended).

### Step 2: Install Claude Code

Install Claude Code globally via npm:

```bash
npm install -g @anthropic-ai/claude-code
```

Or use npx (no installation required):

```bash
npx @anthropic-ai/claude-code
```

### Step 3: Get Your AWS Bearer Token

ADCB will provide you with an `AWS_BEARER_TOKEN_BEDROCK`. This token gives you secure access to Claude via Amazon Bedrock in the `us-west-2` region.

‚ö†Ô∏è **IMPORTANT:**
- **Store your token securely** - you'll need it for configuration
- **Do not share the token outside of your team**

### Step 4: Configure Claude Code for Bedrock

Configure Claude Code using environment variables:

**On macOS/Linux:**

```bash
# AWS Credentials with a Bedrock API Key
export AWS_BEARER_TOKEN_BEDROCK=your-bedrock-api-key

# Enable Bedrock integration
export CLAUDE_CODE_USE_BEDROCK=1
export AWS_REGION=us-west-2

# Using inference profile ID
export ANTHROPIC_MODEL=us.anthropic.claude-sonnet-4-20250514-v1:0

# Recommended output token settings for Bedrock
export CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096
export MAX_THINKING_TOKENS=1024
```

**On Windows (Command Prompt):**

```cmd
# AWS Credentials with a Bedrock API Key
set AWS_BEARER_TOKEN_BEDROCK=your-bedrock-api-key

# Enable Bedrock integration
set CLAUDE_CODE_USE_BEDROCK=1
set AWS_REGION=us-west-2

# Using inference profile ID
set ANTHROPIC_MODEL=us.anthropic.claude-sonnet-4-20250514-v1:0

# Recommended output token settings for Bedrock
set CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096
set MAX_THINKING_TOKENS=1024
```

**On Windows (PowerShell):**

```powershell
# AWS Credentials with a Bedrock API Key
$env:AWS_BEARER_TOKEN_BEDROCK="your-bedrock-api-key"

# Enable Bedrock integration
$env:CLAUDE_CODE_USE_BEDROCK=1
$env:AWS_REGION="us-west-2"

# Using inference profile ID
$env:ANTHROPIC_MODEL="us.anthropic.claude-sonnet-4-20250514-v1:0"

# Recommended output token settings for Bedrock
$env:CLAUDE_CODE_MAX_OUTPUT_TOKENS=4096
$env:MAX_THINKING_TOKENS=1024
```

## Running Claude Code

### Start Claude Code

Navigate to your project directory and launch Claude Code:

```bash
cd your-hackathon-project
claude
```

Or using npx:

```bash
npx @anthropic-ai/claude-code
```

### Verify Connection

Once Claude Code starts, you should see a welcome message. Verify the connection to Sonnet 4 via Bedrock using the `/status` command:

```
/status
```

You should see output similar to:

```
Version: 2.0.33
 Session ID: 0da59082-bdbf-4f5a-9828-XXXXXXXX
 cwd: /Users/UserTest/Labs
 API provider: AWS Bedrock
 AWS region: us-west-2

 Model: us.anthropic.claude-sonnet-4-20250514-v1:0
```

This confirms you're connected to Claude Sonnet 4 through AWS Bedrock in the us-west-2 region.

### Common Commands

- `/help` - Get help with Claude Code features
- `/clear` - Clear the conversation history
- `Ctrl+C` (or `Cmd+C` on Mac) - Exit Claude Code

## Troubleshooting

### Issue: "Authentication Failed" or "Access Denied"

**Solution:**
- Verify your bearer token is correctly copied (no extra spaces or line breaks)
- Check that the token hasn't expired - contact ADCB for a new token
- Ensure the token is in the right location (config.json or environment variable)

### Issue: "Connection Timeout" or "Network Error"

**Solution:**
- Verify you have internet connectivity
- Check that the region in config.json is set to `"us-west-2"`
- Try restarting Claude Code
- Contact hackathon technical support if issues persist

### Issue: Slow Responses

**Solution:**
- This is normal for complex requests - Claude is processing your code
- For faster responses, break large requests into smaller ones
- Check your internet connection speed

### Issue: Claude Command Not Found

**Solution:**
If installed globally:
```bash
npm install -g @anthropic-ai/claude-code
```

Or always use npx (no installation):
```bash
npx @anthropic-ai/claude-code
```

## Token Usage Guidelines

While your bearer token provides access to Claude, be mindful of usage:

- **Optimize prompts**: Be clear and specific to get better responses faster
- **Work incrementally**: Build features step-by-step rather than requesting entire apps
- **Reuse code**: Copy and adapt generated code rather than re-generating similar patterns
- **Share wisely**: Coordinate with your team to avoid duplicate work
